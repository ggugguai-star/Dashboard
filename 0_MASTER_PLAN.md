# 업무 대시보드 — UI 개선 + 위젯 상호작용 2종 — [MODE: R&D]

> ⛔ **0순위 원칙 (협상 불가):** 이번 작업은 **(A) 디자인(UI.md) 개선** + **(B) 아래 '의도된 기능 변경 2개'** 만 한다.
> **그 외 모든 기능·구동·외부연동·창제어·저장은 단 하나도 깨지면 안 된다.** (앱은 현재 "간신히 안정화"된 v3.5.0.)
> **의도된 기능 변경(이 2개만 로직 수정 허용):**
>   1. **위젯 이동 핸들 범위 축소** — 위젯 내부 버튼이 클릭되도록(현재 전체가 드래그 핸들이라 버튼이 안 눌림).
>   2. **위젯 삭제(롱프레스 → 큰 삭제버튼 → 외부클릭 취소)**.
> **그 외 작업 원칙:** CSS 우선. 위 2개를 제외한 JS 로직은 건드리지 않는다. 기존 `id`/`class`/핸들러는 제거·개명 금지(추가만). 의심되면 변경 말고 보고.

## 1. 개요/목표
기존 Tauri v2 "업무 대시보드"를 (A) `UI.md` 디자인 시스템(밝은 톤·iOS Liquid Glass·부드러운 전환/촉감)으로 점진 개선하고, (B) 위젯 **이동 핸들 범위 축소** 및 **삭제 기능(iOS식 롱프레스)** 을 추가한다. 그 외 기능은 100% 보존.

## 2. 성공 기준 (관찰가능)
- (a) UI가 UI.md(글래스·파스텔·타이포·촉감·전환)를 따른다.
- (b) **캘린더 위젯의 오늘/‹/›/+/설정 버튼이 정상 클릭**되고, 위젯 빈 영역 드래그는 정상.
- (c) 위젯을 **롱프레스하면 큰 삭제버튼이 뜨고**, 탭하면 삭제(+실행취소 토스트), 바깥 클릭/ESC로 취소.
- (d) 위 변경 외 **모든 기능(그리드/드래그/스냅·저장복원·Google/Gemini/날씨·창제어)이 그대로 동작** — 기존+신규 테스트 전부 녹색, `cargo check` 통과.

## 3. 범위 & 비목표
- 범위: `src/index.html`(시각/인터랙션) + 기능 2종에 한해 `widget-grid.js`·`store.js`.
- 비목표: 그 외 기능/로직 변경, Electron 잔재 제거, Electron↔Tauri 재마이그레이션, 다크모드.

## 4. 가정
- 런타임 = **Tauri v2**(3.5.0, latest.json=Tauri 업데이터). package.json의 electron-*는 레거시 → 미사용. 빌드=`tauri build`.
- 프론트 = 번들러 없는 순수 HTML/CSS/JS(`frontendDist:"../src"`), `withGlobalTauri:true`. 폰트=로컬 번들(src/fonts).
- 베이스라인 커밋 `0435ab06`(v3.5.0 src + 개선 하네스) 위에서 시작. 단계 diff는 이 HEAD 기준으로 깨끗.

## 5. 기술 스택 (유지)
Tauri v2 + Rust(src-tauri) / **분리 구조**: `src/index.html`(마크업·링크) + `src/styles.css`(CSS) + `src/app.js`·`src/*.js`(JS 모듈).

## 6. 영향 범위 (건드림 / 보존)
- **건드림**: **styles.css(시각 주력)** · index.html(마크업·삭제버튼/토스트 DOM·전환), widget-grid.js(드래그 시작 판정·롱프레스 감지 — 기능 2종 한정), store.js(위젯 제거 한 군데 — 삭제 기능 한정).
- **보존(절대 변경 금지)**: layout-engine.js, google-api/gemini/weather/google-cache/token-store/local-widgets, store의 그 외 로직·데이터 포맷, 모든 DOM id/구조·이벤트 계약, IPC(window.__TAURI__), 창제어(프레임리스/드래그/단축키/자동시작).

## 7. 핵심 계약 / 제약 (먼저 고정)
- **CSP 준수**: style/script/font-src 가 `'self' tauri://localhost` 한정 → **CDN 폰트/CSS 금지**, 로컬 @font-face만. 외부 URL 추가 금지.
- **프레임리스 보존**: `decorations:false` → `data-tauri-drag-region`·창 컨트롤 유지.
- **단계 diff < 12000자**(파일 분리했어도 styles.css 가 크므로 섹션 단위로 잘게). 단계당 파일 변경 최소.
- **회귀 계약**: 기존 `*.test.mjs` + 신규 행동 테스트 전부 녹색이 매 단계 DoD.
- **기능 2종 구분 임계값(기본값, 조정 가능)**: 드래그=이동 6px 초과 / 롱프레스=550ms 정지 유지. 둘은 상호배타(움직이면 드래그, 멈춰 길게 누르면 삭제모드).

## 8. 비기능 요구
성능(인라인 CSS 비대화 주의·transform/opacity 애니메이션), 접근성(대비4.5:1·reduced-motion/transparency·focus-visible·삭제 ESC), 경량 유지(~7MB).

## 9. 위험 & 완화 (회귀 위험 포함)
1. 거대 단일 파일 회귀 → 시각만 바꾸고 DOM/핸들러 불변, 단계별 행동 테스트.
2. CSP 위반 폰트/스타일 미로드 → 로컬 자원만, 변경 후 스모크로 콘솔 에러 확인.
3. 드래그 vs 롱프레스 vs 클릭 충돌 → 임계값/상태머신 명확화 + 테스트(드래그 시 삭제모드 안 뜸, 버튼 클릭이 드래그로 안 샘).
4. 삭제 실수로 위젯 유실 → **실행취소 토스트(5초)** + store 변경은 삭제 1군데로 국소화.
5. cursor-agent 타임아웃(전적 있음) → 단계를 작게 + 오케스트레이터 `CURSOR_CLI_TIMEOUT=3600` 상향 완료.

## 10. 단계별 구현 계획 (기능 먼저 → 시각, 위험 격리)

### [단계 1] 베이스라인 + 행동 특성화 테스트 + 게이트 보정
- 목적: v3.5.0 고정 + 회귀 안전망 + 이 프로젝트용 게이트 보정.
- 작업: 현 `cargo check`·`*.test.mjs` 통과 확인(무변경). 드래그/스냅·저장복원·위젯 추가 등 **핵심 행동 특성화 테스트** 보강. `scripts/static_check.py`에서 `BLOCK_NPM_BUILD=False`(프론트 빌드 없음·electron-builder 오발 방지).
- DoD: cargo check 통과 / 전체 테스트 녹색 / electron-builder 미실행 / 기능 변경 0.

### [단계 2] (기능1) 위젯 이동 핸들 범위 축소
- 목적: 위젯 내부 버튼이 정상 동작하도록 드래그 시작 영역 축소.
- 작업: widget-grid.js의 드래그 시작(pointerdown) 판정에 **인터랙티브 요소 제외 가드** 추가 — `button, a, input, select, textarea, [contenteditable], [role="button"], .no-drag`(및 스크롤 영역)에서 시작한 포인터는 드래그로 인식하지 않음. 필요 시 index.html에 `.no-drag` 부착(추가만).
- DoD: **캘린더 오늘/‹/›/+/설정 버튼 클릭됨**, 빈 영역 드래그 정상, 모든 위젯 공통. 드래그/스냅/저장 회귀 없음(테스트).

### [단계 3] (기능2) 위젯 삭제 — 롱프레스 → 삭제버튼 → 취소
- 목적: iOS식 삭제 상호작용 추가.
- 작업: widget-grid.js에 롱프레스 감지(550ms 정지, 드래그와 상호배타). 삭제모드 진입 시 위젯에 **큰 삭제 버튼** 오버레이(이 단계는 동작 우선, 시각은 후속 단계서 UI.md화). 탭 → store에서 위젯 제거 + **실행취소 토스트**. 활성 위젯 외 클릭/ESC → 취소.
- 변경: widget-grid.js(감지/취소), store.js(제거 1군데), index.html(삭제버튼·토스트 DOM).
- DoD: 롱프레스로 삭제버튼 등장→탭 삭제→토스트 복원 가능, 외부클릭/ESC 취소, **드래그와 충돌 없음**. 그 외 회귀 없음(테스트).

### [단계 4] 전역 디자인 토큰 + 로컬 폰트

- DoD: 토큰/폰트 로컬 로드(콘솔 CSP 에러 0), 화면 깨짐 없음, 테스트 녹색, cargo check.
